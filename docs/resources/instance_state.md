# zstack_instance_state (Resource)

Manages the power state of an existing ZStack VM instance. This resource only starts or stops the VM. Removing this resource from Terraform configuration stops managing the VM power state and does not delete, start, or stop the VM.

## Example Usage

```terraform
# Copyright (c) ZStack.io, Inc.

resource "zstack_instance_state" "test" {
  vm_instance_uuid = zstack_instance.test.uuid
  state            = "Running"
}
```

## Schema

### Required

- `state` (String) Desired VM power state. Supported values are `Running` and `Stopped`.
- `vm_instance_uuid` (String) UUID of the ZStack VM instance whose power state is managed.

### Optional

- `operation_timeout` (Number) Maximum number of seconds to wait for the VM to reach the desired state. Defaults to `600`.
- `stop_type` (String) Stop mode used when `state` is `Stopped`. Currently only `grace` is supported. Defaults to `grace`.

### Read-Only

- `id` (String) Same as `vm_instance_uuid`. Used for Terraform tracking.
