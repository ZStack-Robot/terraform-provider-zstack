# zstack_log_server (Resource)

Manage log server in ZStack.

Use `appender_type` and `appender_configuration` for normal Terraform authoring. The provider converts them to the nested JSON required by ZStack, for example `{"appenderType":"Syslog","configuration":{...}}`.

The raw `configuration` attribute is kept for compatibility. When using it directly, it must already be the nested JSON object with `appenderType` and `configuration`; a flat object such as `{"host":"127.0.0.1","port":9200}` is not valid for the ZStack API.

## Example Usage

```terraform
# Copyright (c) ZStack.io, Inc.

resource "zstack_log_server" "example" {
  name          = "example-log-server"
  category      = "ManagementNodeLog"
  type          = "Log4j2"
  level         = "INFO"
  appender_type = "Syslog"
  appender_configuration = {
    hostname = "logs.example.com"
    port     = "514"
    protocol = "UDP"
    facility = "LOCAL5"
  }
}

output "zstack_log_server" {
  value = zstack_log_server.example
}
```

## Schema

### Required

- `category` (String) The log category.
- `name` (String) The name of the log server.
- `type` (String) The log server type.

### Optional

- `appender_configuration` (Map of String) Appender configuration key/value map used to build the nested configuration JSON. For Syslog, typical keys are hostname, port, protocol, and facility.
- `appender_type` (String) Appender type used to build configuration, for example Syslog. Must be set with appender_configuration and omitted when configuration is set.
- `configuration` (String) Raw log server configuration JSON. Must be a JSON object containing appenderType and configuration. Mutually exclusive with appender_type/appender_configuration.
- `description` (String) The description of the log server.
- `level` (String) The log level.

### Read-Only

- `uuid` (String) The UUID of the log server.
